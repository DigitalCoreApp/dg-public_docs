# Paid Connection

The paid connection lets your AI assistant read and write your live DigitalCore data. It needs an API key.

## What you can do

- Ask about engagements, plans, and actuals.
- Record monthly check-in numbers.
- Generate Responses (option sets) and apply them.
- Open Decisions and update them.
- Draft executive summaries.

## What is needed

- A DigitalCore account.
- An [API key](/mcp-and-integrations/api-keys) on a paid plan.
- An AI client that supports MCP (Claude Desktop, Cursor, others).

## Setup

The endpoint is `https://mcp.digitalcore.app`.

### Claude Desktop

1. Open Settings, then Connectors.
2. Add a connection.
3. URL: `https://mcp.digitalcore.app`.
4. Authentication: API key.
5. Paste your key.
6. Save and restart.

### Cursor

1. Open Cursor settings, then MCP.
2. Add server.
3. URL: `https://mcp.digitalcore.app`.
4. Add the API key as an authorization header.

### Other clients

The connection follows the MCP standard. Provide the URL and the API key as a bearer token.

## What is exposed

The paid connection exposes the same actions you use in the app: monitor, record, diagnose, plan, decide, communicate. Your role and the API key''s access level decide what is allowed.

## Limits

- Rate limits apply. The default is 60 requests per minute per key.
- Writes are logged in the activity log under your name.
- Read-only keys cannot record actuals or apply Responses.

## Tips

- Start with a read-only key to test.
- Once it works, create a separate read-and-write key for daily use.
- Rotate keys every 6 months.
