# Platform Reference

This section follows the live application navigation. Use it as the manual for any page in the product.

## The seven workspaces

DigitalCore is organised into seven workspaces. The first five are where you work day to day. The last two are where you set things up.

| Workspace | What it is for |
|-----------|----------------|
| [Pulse](/platform/pulse) | Your daily landing screen. Signals, attention items, and recent activity. |
| [Portfolio](/platform/portfolio) | Cross-engagement view of margin, cost, capacity, and risk. |
| [Engagements](/platform/engagements) | Manage one engagement: check-ins, health, projects, risks. |
| [Decisions](/platform/decisions) | Cases, priorities, AI-generated Responses, and the response loop. |
| [Communicate](/platform/communicate) | Reports, client portals, summaries, notifications. |
| [Configure](/platform/configure) | Services, contracts, templates, library, AI agents, thresholds, policies. |
| [Organization](/platform/organization) | Users, partners, stakeholders, roles. |

## How the workspaces connect

```
Configure        rules of the game (services, templates, agents, thresholds, policies)
   v
Engagements      where data is recorded (check-ins, hours, KPIs, finance actuals)
   v
Pulse + Portfolio  what the data means (signals, health, economics)
   v
Decisions        what to do about it (cases, options, approve & apply)
   v
Communicate      who needs to know (reports, portals, notifications)
```

Most operators live in Pulse. Strategy and finance leaders live in Portfolio. Delivery managers live in Engagements. Admins set things up in Configure and Organization.
