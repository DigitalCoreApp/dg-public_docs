# Compare

Compare puts two or more engagements next to each other so you can see, line by line, where they differ.

## What you compare

| Dimension | Detail |
|-----------|--------|
| Margin | Revenue, cost, margin percent, with trend. |
| Capacity | Planned versus actual hours, leakage rate, role mix. |
| Performance | KPI achievement, breach count, breach value. |
| AI Usage | Token spend, model split, ROI per use case. |
| Health | Composite health score and its components. |

## When to use it

- **Renewal pricing.** Does this engagement need a different rate than its peers?
- **Practice benchmarking.** Which delivery teams perform best on similar work?
- **Lessons learned.** Why does engagement A run hot while engagement B does not?
- **Scope similarity.** Like-for-like reads on services using the same templates.

## Filters and grouping

Pick engagements one by one, or filter by:

- Service, portfolio, or [tag](/platform/configure/service-setup/service-tags).
- Customer.
- Contract type (fixed, time and materials, outcome).
- Period.

Group by any of the above to see averages and outliers.
