# Users

This page is where you manage everyone with login access to DigitalCore.

## User fields

| Field | Detail |
|-------|--------|
| Name | Display name. |
| Email | The address used to sign in. |
| Role | Admin, Manager, Operator, or Viewer. See [App Roles](./app-roles). |
| Status | Invited, Active, or Deactivated. |
| Multi-factor | On or off. |
| Last sign-in | When they last signed in. |
| Engagements | Where they are owner or contributor. |

## Invite a user

Admins click **Invite User**, enter an email and role, and the new user gets an invitation email. They set their password the first time they sign in.

## Lifecycle

| Action | Effect |
|--------|--------|
| Invite | Sends an email. Status: Invited. |
| Activate | Happens on first sign-in. |
| Deactivate | Removes access. Their data and history stay. |
| Re-activate | Restores access. |
| Delete | Admin-only and rare. History is preserved as anonymised. |

## Activity log

Every user action is recorded: sign-ins, recordings, decisions, and configuration changes. You can view it per user or for the whole organisation.
