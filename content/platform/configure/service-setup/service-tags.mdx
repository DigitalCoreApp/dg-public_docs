# Service Tags

Tags are free-form labels you attach to services and engagements. [Portfolios](./portfolios) are formal groupings. Tags are flexible: a service can carry many, and you can create them on the fly.

## When to use tags

| Need | Example |
|------|---------|
| Theme | growth-account, at-risk |
| Technology | aws, azure, microsoft-365 |
| Customer segment | public-sector, mid-market |
| Initiative | q2-2026-rollout |
| Compliance | gdpr-in-scope, iso27001 |
| Maturity | pilot, production, sunset |

Tags appear as filters on [Portfolio](/platform/portfolio), [Pulse: Signals](/platform/pulse/signals), and [Engagements](/platform/engagements).

## Keeping tags tidy

The Tags screen shows:

- Top tags by usage.
- Orphan tags used by zero or one entity.
- Suggested merges for similar spellings.

Admins can rename or merge tags. Existing usage updates automatically.

## Tags or portfolios?

| Use | When |
|-----|------|
| Portfolio | You need a formal rollup of health and margin. |
| Tags | You need flexible filtering and themes. |

Use both. They are not exclusive.
