# Templates

Templates are the reusable link between an engagement and the [Library](/platform/configure/data/library). Pick a template when you start an engagement and you get the same structure across every engagement of that service.

## Four template types

| Template | What it tracks |
|----------|----------------|
| Finance | P&L line items. |
| Performance | KPIs and service terms. |
| Capacity | Roles and effort. |
| AI Usage | AI use cases and cost. |

Every template item points to a library entry. That is what makes numbers comparable across engagements.

## What a template item looks like

A Finance template item, for example:

| Field | Example |
|-------|---------|
| Library entry | Labour Cost. |
| Default planned value | 0, or your starting forecast. |
| Period | Monthly, quarterly, or annual. |
| Default delivery party | Customer, deliverer, or partner. |
| Active periods | Months in scope. |

Capacity items point to roles, and each role has a Finance destination, so recorded hours post as labour cost automatically.

## Template, policy, plan, actual

| Stage | Where it lives | Purpose |
|-------|----------------|---------|
| Template | Organisation | Reusable list of items. |
| Policy | Engagement | Per-item settings, like default delivery party. |
| Plan | Per period | Target or forecast. |
| Actual | Per period | What you recorded. |

Admins manage templates and policies. Operators record actuals during check-ins.

## Default templates per service

A [service](./services) can pre-select templates so a new engagement starts with the right Finance, Performance, Capacity, and AI structure in one click.
