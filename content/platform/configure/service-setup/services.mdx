# Services

A service is something you deliver: Application Support, Cloud Migration, Managed Security. Most other things in DigitalCore attach to a service or to one of its [engagements](/platform/engagements).

## Service fields

| Field | Purpose |
|-------|---------|
| Name | Short label, for example "Cloud Operations". |
| Category | Grouping such as Managed Service, Project, or Advisory. |
| Owner | The person accountable for the service. |
| Description | What is included and what is not. |
| Default templates | [Templates](./templates) used when you start a new engagement. |

## Service versus engagement

A service is the catalog entry. An engagement is one instance of that service for a customer or internal team.

| Service | Engagement |
|---------|------------|
| Cloud Operations | Cloud Operations for Acme Corp |
| Application Support | App Support for Internal Finance |
| Defined once | Created per customer |

Reports roll up at both levels. See [Portfolio](/platform/portfolio).

## Lifecycle

1. Draft. Service exists but is not yet active.
2. Active. You can create engagements against it.
3. Deprecated. No new engagements; existing ones continue.
4. Archived. Fully retired.
