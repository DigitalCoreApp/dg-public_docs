# Projects & Initiatives

Some engagements are projects with a fixed scope and end date. Others are services that run continuously. The Projects view is for the first kind.

## How projects differ from services

| Dimension | Service | Project |
|-----------|---------|---------|
| Duration | Open-ended. | Fixed start and end. |
| Tracking | Period-based, monthly KPIs. | Milestone-based. |
| Margin | Per-month. | Earned value over total. |
| Closure | Status changes to Closed. | Final deliverable plus retrospective. |

Projects use the same engagement entity and the same templates. They just present differently here.

## What you see in the project view

- Phases and milestones with target and actual dates.
- Percent complete based on milestones hit.
- Earned value: planned spend times percent complete versus actual spend.
- Burn rate: hours or spend per week.
- Forecast end date based on burn.

## When to use this view

Use the Projects page when the engagement has:

- A defined end date.
- Discrete deliverables or milestones.
- Fixed-fee or hybrid pricing.
- A retrospective at the end.

For ongoing managed services, [Engagement Overview](/platform/engagements/overview) is usually the better lens.
