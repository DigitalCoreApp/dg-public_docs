# Your First Check-in

A check-in is the monthly entry of actuals on an engagement. It produces variance, missed SLA terms, labour cost, utilisation, and an updated health score.

Go to **Engagements**, open your engagement, and click **Check-in**.

## Enter finance actuals

The Finance tab shows your template items as rows and months as columns. Each cell shows the plan and an input for actual.

1. Type the actual amount for each item.
2. The cell colours green, amber, or red based on variance.

Two lines fill in automatically:

- **Labour cost**, when capacity hours are saved.
- **SLA penalties**, when a performance miss is detected.

These rows show a source tag so you know they came from another tab, not manual entry.

## Enter performance actuals

Open the Performance tab. Type the actual value for each KPI.

If a value misses the contract SLA term, DigitalCore:

1. Detects the miss using the term''s operator.
2. Calculates the penalty (fixed, percentage of contract value, or tiered).
3. Records a penalty event for the activity log.
4. Adds a finance entry for the penalty if the contract calls for it.

## Enter capacity hours

Open the Capacity tab. Type the actual hours per role for the month.

DigitalCore picks the right rate (contract override, rate card, or default), multiplies hours by rate, and posts the labour cost to your finance grid.

## Review the results

- **Summary cards** at the top show finance, performance, and capacity totals.
- **Health colour** (Green, Amber, Red) is set per item, then rolls up to the domain and engagement.
- **Cross-domain effects**: capacity hours land as finance cost, missed terms land as finance penalty.

## Autonomous agents start working immediately

The moment a check-in is saved (or any other data lands — see options below), DigitalCore's autonomous agents pick it up. You do not need to click anything.

- **Aggregation rollups** refresh so dashboards and pulse views reflect the new month.
- **Trend narratives** are generated for the engagement and the portfolio.
- **Cross-domain correlation scans** look for signals such as margin pressure, cost escalation, capacity overrun, AI-budget overrun, and SLA breach.
- **Health scores** recompute at item, domain, and engagement level.
- **Governance cases open automatically** for critical signals — with the matching risks, issues, decisions, and follow-ups already populated. See [Decision Cases](/platform/decisions/cases).
- **Notifications fire** to the channels you have configured (in-app, email, Slack, Teams). See [Notification Channels](/mcp-and-integrations/notification-channels).

## Don't want to type? Other ways to load data

The check-in screen is the fastest path for a single month. For everything else there is a faster route:

| Path | Best for |
|------|----------|
| [Excel / CSV import](/mcp-and-integrations/excel-import) | Backfilling history, finance-team workflows, bulk catch-up across many months. |
| [QuickBooks](/mcp-and-integrations/quickbooks) | Continuous Finance actuals synced from your accounting system. |
| [ServiceNow](/mcp-and-integrations/servicenow) | Continuous Performance actuals (incidents, SLA results) from ITSM. |
| [DigitalCore MCP](/mcp-and-integrations/paid-mcp) | Recording values from an AI assistant such as Claude, Cursor, or ChatGPT. |

All four paths feed the same plans and trigger the same autonomous agents. Mix and match per domain.
